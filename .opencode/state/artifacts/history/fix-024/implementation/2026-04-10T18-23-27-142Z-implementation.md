# FIX-024 Implementation — Response Envelope Stripping and Path-Mode Search

## Summary
Fixed two confirmed production defects in the node-client response handling and path-mode search output parsing.

---

## Defect 1: HubNodeClient Response Envelope Stripping

### Root Cause
HubNodeClient methods (`read_file`, `search`, `git_status`, `list_directory`) were extracting only the inner `data` dict from the OperationResponse envelope:
```python
return data.get("data", {})  # strips {"success": True, "data": {...}} → {"entries": [...]}
```

Hub handlers then called `result.get("success", False)` which always returned `False` because the stripped payload has no `success` key.

### Changes Made

#### File: `src/hub/services/node_client.py`

**read_file method (line 206-208):**
```python
# Before:
if data.get("success"):
    return data.get("data", {})

# After:
return data  # full OperationResponse: {"success": True, "data": {...}}
```

**search method (line 288-290):**
```python
# Before:
if data.get("success"):
    return data.get("data", {})

# After:
return data  # full OperationResponse: {"success": True, "data": {...}}
```

**git_status method (line 318-320):**
```python

```

**list_directory method (line 347-349):**

#### File: `src/hub/tools/inspection.py`

**read_repo_file_handler (lines 290-296):**

## Defect 2: Path-Mode Search Output Parsing

### Root Cause
`executor.py` search_files() uses `--files-with-matches` for `mode="path"` (line 210), which outputs just filenames (one per line, no colons). But the parsing loop (lines 254-263) assumed `filename:line_number:content` format:
```python
parts = line.split(":", 2)
int(parts[1])  # ValueError for "foo.py" — no colon
```

### Changes Made

#### File: `src/node_agent/executor.py`

**search_files method (lines 245-294):**
```python
# Before:
for line in output.strip().split("\n"):
    if not line:
        continue
    parts = line.split(":", 2)
    if len(parts) >= 2:
        file_path = parts[0]
        try:
            line_num = int(parts[1])
        except ValueError:
            line_num = 0
        ...

# After:
# Path-mode uses --files-with-matches which outputs just filenames (no colons)
if mode == "path":
    for line in output.strip().split("\n"):
        line = line.strip()
        if not line:
            continue
        files_searched.add(line)
        # For path mode, each line is just a filename with no line number or content
        if len(matches) < max_results:
            matches.append(
                {
                    "file": line,
                    "line_number": 0,
                    "match_count": 0,
                    "content": "",
                }
            )
else:
    for line in output.strip().split("\n"):
        if not line:
            continue
        parts = line.split(":", 2)
        if len(parts) >= 2:
            file_path = parts[0]
            try:
                line_num = int(parts[1])
            except ValueError:
                line_num = 0
            ...
```
