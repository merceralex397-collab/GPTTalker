# Code Review for FIX-020: Authentication Enforcement on Node Agent Operational Routes

## Verdict: APPROVED

## Summary
The implementation correctly adds API key authentication enforcement to all 5 node agent operational routes. The `require_api_key` dependency is properly implemented, applied in the correct dependency order, and handles both the "auth not configured" and "auth required but missing/invalid" cases.

## Review Questions

### 1. Is the `require_api_key` dependency correctly implemented?

**YES.** The dependency at `src/node_agent/dependencies.py:25-43`:

- Uses proper FastAPI async dependency pattern: `async def require_api_key(request: Request, config: NodeAgentConfig = Depends(get_config)) -> None`
- Correctly accesses `config.api_key` which is defined in `NodeAgentConfig` at `config.py:17` as `str | None`
- Returns early (no-op) when `config.api_key is None` — allowing requests when auth is not configured
- Extracts and validates Bearer token from `Authorization` header
- Raises `HTTPException(status_code=401, detail="Missing or invalid Authorization header")` when header is absent or doesn't start with "Bearer "
- Raises `HTTPException(status_code=401, detail="Invalid API key")` when token doesn't match configured key

### 2. Is auth applied in the correct order (before `get_executor`)?

**YES.** In all 5 routes, `require_api_key` is declared before `get_executor`:

```python
async def list_dir(
    request: ListDirRequest,
    _: None = Depends(require_api_key),       # ← auth first
    executor: OperationExecutor = Depends(get_executor),  # ← executor second
) -> OperationResponse:
```

FastAPI evaluates `Depends()` parameters in declaration order, so `require_api_key` runs before `get_executor`, ensuring unauthenticated requests are rejected before any executor operations occur.

### 4. Does the auth skip correctly when `api_key is None`?

**YES.** Lines 33-35 in `dependencies.py`:

```python
if config.api_key is None:
    # Auth not configured - allow all requests (node is open)
    return
```

When `api_key` is `None` (the default), the function returns immediately without raising any exception, effectively making the node agent open (no auth required). This is the correct fail-open behavior for when authentication is intentionally not configured.

**YES.** Two separate checks:

- **Missing/invalid format** (lines 37-39):

  ```python
  if token != config.api_key:

  ```

### 6. Are imports correct?

**YES.** `operations.py:11`:

All required imports (`FastAPI`, `Depends`, `HTTPException`, `Request` from `fastapi`; `NodeAgentConfig` from local config; `OperationExecutor` from local executor) are properly imported in `dependencies.py`.
