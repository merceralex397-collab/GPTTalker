## Defect Addressed in This Remediation

**Defect 5: MCP error double-wrapping in `src/hub/transport/mcp.py`**

When `_execute_handler` returns `{"success": False, "error": "..."}`, the original `format_tool_response` went to the success path (no `error` param passed), wrapping the error dict inside `result.data`, producing malformed JSON-RPC:
```json
{"jsonrpc": "2.0", "id": 1, "result": {"success": false, "data": {"success": false, "error": "..."}}}
```

## Fix Applied

Added error-promotion check in `format_tool_response` before the success-path handling:

```python
if isinstance(result, dict) and result.get("success") is False and "error" in result:
    response["error"] = {
        "code": -32603,
        "message": result.get("error", "Unknown error"),
    }
    return response
```
