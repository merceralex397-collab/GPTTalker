# Planning Artifact for FIX-020

## Ticket
- **ID**: FIX-020
- **Title**: Fix missing authentication enforcement on node agent operational routes
- **Wave**: 14
- **Lane**: bugfix

## Root Cause
The 5 operation routes in `src/node_agent/routes/operations.py` accept requests without any authentication check. `NodeAgentConfig.api_key` (line 17 of `config.py`) is defined but never enforced. Any client can trigger node operations without credentials.

The hub sends `Authorization: Bearer <api_key>` via `NodeAuthHandler` but the node agent never validates it.

## Fix Approach
Add a `require_api_key` async FastAPI dependency in `src/node_agent/dependencies.py` that:
- Skips validation when `api_key is None` (auth not configured, open node)
- Returns `401 Unauthorized` when auth is configured but bearer token is missing or invalid
- Uses the existing `get_config` pattern to access `NodeAgentConfig.api_key` from app state

Then add `Depends(require_api_key)` to each of the 5 routes in `operations.py`.

## Implementation Steps

### Step 1: Add `require_api_key` dependency to `src/node_agent/dependencies.py`
Create an async dependency function that:
- Receives `request: Request` and `config: NodeAgentConfig = Depends(get_config)` as parameters
- When `config.api_key is None`: returns None (auth not configured - allow all)
- When `config.api_key` is set: checks `Authorization` header for `Bearer <token>`, returns 401 if missing or wrong token

Also need to add `HTTPException` import from `fastapi`.

### Step 2: Apply auth to all 5 routes in `src/node_agent/routes/operations.py`
Add `require_api_key` to the `Depends()` chain of each route:
- `list_dir`, `read_file`, `search`, `git_status`, `write_file`: add `Depends(require_api_key)` before the existing `Depends(get_executor)`

The `Depends(require_api_key)` must come **before** `Depends(get_executor)` in the chain so auth is checked first.
