## Background

FIX-020 is a security remediation ticket reopened via `issue_intake`. The original defect was missing authentication on node-agent operational routes. Investigation identified that Defects 1–4 and 6–7 were already correctly fixed in the current codebase. Only Defect 5 (MCP error double-wrapping in `src/hub/transport/mcp.py`) required a new code fix, which was applied.

**Defect 5 fix:** Added error-promotion check in `format_tool_response()` to promote handler error dicts to proper JSON-RPC 2.0 error responses, preventing errors from being buried inside `result.data`.

## Defect 5 Fix Verification

**File:** `src/hub/transport/mcp.py`  
**Function:** `format_tool_response()` (lines 103–162)

**Applied fix (lines 137–143):**
```python
# If handler returned an error dict, promote it to proper JSON-RPC error
if isinstance(result, dict) and result.get("success") is False and "error" in result:
    response["error"] = {
        "code": -32603,
        "message": result.get("error", "Unknown error"),
    }
    return response
```

This check fires before the success-path handling (lines 145–157), ensuring that handler error dicts are promoted to proper JSON-RPC 2.0 error responses instead of being buried inside `result.data`.

**Result: PASS** — Logic correctly placed before success-path handling.
