# FIX-026 Planning: Fix Missing Node Health Hydration at Startup

## Problem Statement

When nodes are registered via `NodeRepository.create()`, health columns are inserted as **NULL**. The hub's `lifespan.py` startup calls `mcp_handler.initialize()` but **never calls `node_health_service.check_all_nodes()`** to hydrate health data. This causes:

1. `NodePolicy.validate_node_access()` queries health via `get_node_health()`
2. `get_node_health()` finds NULL → returns `NodeHealthStatus.UNKNOWN`
3. Policy sees UNKNOWN → returns `"unknown_health_status"` rejection
4. All node-scoped tools (list_repos, git_status, inspect_repo_tree, search_repo) fail

**Note**: FIX-025 correctly wired `NodeHealthService` into `NodePolicy` (fixing the `NoneType` error), but did not address the missing startup hydration call.

## Root Cause

Two-part issue:
1. `MCPProtocolHandler.initialize()` creates `node_health_service` but does not store it as an instance attribute
2. `lifespan.py` never calls `check_all_nodes()` after MCP initialization

## Implementation Plan

### Step 1: Store `node_health_service` in `MCPProtocolHandler` (`src/hub/mcp.py`)

After line 99 (where `node_health_service` is created), store it as an instance attribute:

```python
# After line 99, add:
self._node_health_service = node_health_service
```

This allows `lifespan.py` to access the service after initialization.

### Step 2: Call `check_all_nodes()` in `lifespan.py` (`src/hub/lifespan.py`)

After line 139 (`await mcp_handler.initialize(app)`), add health hydration:

```python
# Step 9: Hydrate node health before accepting traffic (FIX-026)
# This populates health columns so NodePolicy can make fail-closed decisions
try:
    if hasattr(mcp_handler, '_node_health_service') and mcp_handler._node_health_service is not None:
        await mcp_handler._node_health_service.check_all_nodes()
        logger.info("node_health_hydrated_at_startup")
except Exception:
    # Fail-open: log error but don't crash the hub
    logger.exception("node_health_hydration_failed_at_startup")
```
