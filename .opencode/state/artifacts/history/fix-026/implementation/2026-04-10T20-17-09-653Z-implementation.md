# FIX-026 Implementation: Fix missing node health hydration at startup causing policy denials

## Summary
Implemented Step 8b in `src/hub/lifespan.py` to hydrate node health status for all registered nodes at startup, before the hub accepts traffic. This fixes the issue where `NodePolicy.validate_node_access()` would see UNKNOWN health status and fail-closed reject all node-scoped tool calls.

## File Changed
`src/hub/lifespan.py` (lines 142-159 added)

## Implementation

### Before (after `mcp_handler.initialize()` at line 140):
```python
    logger.info("mcp_handler_initialized")

    # Yield control to the application
    yield
```

### After (Step 8b inserted before `yield`):
```python
    logger.info("mcp_handler_initialized")

    # Step 8b: Hydrate node health for all registered nodes before accepting traffic
    # (fail-open: log warning but continue if health check fails)
    try:
        from src.hub.services.node_health import NodeHealthService

        node_health_service = NodeHealthService(
            http_client=app.state.http_client,
            node_repo=app.state.db_manager._repos.node,
            auth_handler=None,
        )
        await node_health_service.check_all_nodes()
        logger.info("node_health_hydrated")
    except Exception as e:
        logger.warning(
            "node_health_hydration_failed_continuing",
            error=str(e),
            exc_info=e,
        )

    # Yield control to the application
    yield
```

## Key Design Decisions

1. **Placement**: Step 8b runs after `mcp_handler.initialize()` (Step 8) but before `yield`, ensuring the MCP router is fully built before health hydration begins.

2. **Fail-open pattern**: If health hydration fails, the hub starts anyway with a warning logged. This prevents a single node's health check failure from blocking the entire hub startup.

3. **Uses existing `db_manager._repos.node`**: Rather than constructing a new `NodeRepository` instance, the implementation accesses the already-initialized node repository via `app.state.db_manager._repos.node`. This is consistent with how other services access the repository during startup.

4. **auth_handler=None**: Internal hub-to-node health checks use no authentication (the health endpoint on node agents does not require auth per FIX-020).
