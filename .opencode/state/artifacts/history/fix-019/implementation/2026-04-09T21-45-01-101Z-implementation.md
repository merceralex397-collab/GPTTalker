# FIX-019: Implementation - Fix MCP tools endpoint returning empty list

## Summary

Moved tool registration from deprecated `@app.on_event('startup')` into the lifespan context manager where it will actually execute.

## Changes Made

### 1. `src/hub/lifespan.py` — Added Step 8a

Added tool registration BEFORE `mcp_handler.initialize(app)` in the lifespan context manager:

```python
    # Step 8a: Register all MCP tools BEFORE initializing MCP handler
    # Note: @app.on_event("startup") in main.py is deprecated when lifespan= is set.
    # Tools must be registered before mcp_handler.initialize() builds the router.
    from src.hub.tool_router import get_global_registry
    from src.hub.tools import register_all_tools

    registry = get_global_registry()
    register_all_tools(registry)
    logger.info("mcp_tools_registered", tool_count=registry.tool_count)
```

This step now runs at lifespan startup (before `yield`) and ensures all MCP tools are registered on the global registry before the `PolicyAwareToolRouter` is built by `mcp_handler.initialize(app)`.

### 2. `src/hub/main.py` — Removed deprecated handler

Removed:
- The orphaned `@app.on_event("startup")` handler (former lines 57-64)
- The unused `from src.hub.tool_router import get_global_registry` import (line 9)

The app now cleanly ends at line 53 with `app.include_router(router)` — no startup event handlers remain.

## Why This Fix Works

1. **Root cause**: FastAPI's `@app.on_event("startup")` is deprecated when `lifespan=` is set on the app. The lifespan context manager takes over lifecycle management, and the old event-based handlers do not fire.

2. **Fix**: The lifespan context manager (`src/hub/lifespan.py`) is the proper place for startup logic when using the modern FastAPI pattern. By adding `register_all_tools(registry)` as Step 8a, tools are registered before `mcp_handler.initialize()` builds the router.

3. **Timing**: The registry is obtained via `get_global_registry()` at Step 8a, then tools are registered via `register_all_tools(registry)`. When `mcp_handler.initialize(app)` runs at Step 8, it passes the same registry to `PolicyAwareToolRouter`, which will now contain all registered tools.
