# Plan: FIX-022 — Fix HubNodeClient.read_file HTTP method and endpoint mismatch

## Bug Description

`HubNodeClient.read_file` in `src/hub/services/node_client.py` (line 195) uses:
```python
response = await self.get(node, f"/files/read?path={path}", timeout=30.0)
```

This calls `GET /files/read?path={path}` — but the node agent only exposes `POST /operations/read-file` (line 136 of `src/node_agent/routes/operations.py`) with a JSON body.

Additionally, `GET /files/read` does not support `offset`/`limit` parameters, while the node agent's `ReadFileRequest` accepts:
- `path: str` (from `OperationRequest` base)
- `offset: int = 0`
- `limit: int | None = None`

This contract mismatch means `read_file` tool calls always fail.

## Fix

Change `HubNodeClient.read_file` to use `POST /operations/read-file` with a JSON body.

### Method Signature Change

Before:
```python
async def read_file(
    self,
    node: NodeInfo,
    path: str,
) -> dict[str, Any]:
```

After:
```python
async def read_file(
    self,
    node: NodeInfo,
    path: str,
    offset: int = 0,
    limit: int | None = None,
) -> dict[str, Any]:
```

### Request Call Change

Before (line 195):
```python
response = await self.get(node, f"/files/read?path={path}", timeout=30.0)
```

After:
```python
response = await self.post(
    node,
    "/operations/read-file",
    json={"path": path, "offset": offset, "limit": limit},
    timeout=30.0,
)
```

### Import Change

No new imports needed — `HubNodeClient` already has `post()` method (line 121).
