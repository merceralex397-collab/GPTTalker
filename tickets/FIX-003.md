# FIX-003: Fix hub MCP router async wiring and circular import

## Summary

Three related hub wiring issues: (1) MCPProtocolHandler._ensure_router() calls loop.run_until_complete() which fails inside an already-running async event loop, making all MCP endpoints raise RuntimeError. (2) src/hub/main.py imports router from routes.py while routes.py imports mcp_handler from main.py, creating a circular import. (3) Duplicate /health endpoints exist in both main.py and routes.py. Refactor to use FastAPI dependency injection for the MCP handler, break the circular import, and remove the duplicate endpoint.
