# FIX-005: Fix structured logger TypeError and HubConfig attribute error

## Summary

Two related logging/config bugs: (1) Multiple modules call logger.info(..., key=value) with arbitrary kwargs, but get_logger() returns a standard logging.Logger which raises TypeError on those calls. Either use a custom logging adapter that handles extra kwargs or pass structured data via the extra dict. (2) Hub lifespan.py references config.database_path but HubConfig only defines database_url. Fix the attribute reference to match the config model.
