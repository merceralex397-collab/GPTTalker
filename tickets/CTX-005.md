# CTX-005: Context bundle generation

## Summary

Implement the build_context_bundle MCP tool that assembles task-type-aware context bundles for ChatGPT. Different task types (debugging, feature development, code review, documentation) require different context compositions. The bundle builder prioritizes and selects content to fit within token-limited contexts.

## Acceptance Criteria

- [ ] build_context_bundle MCP tool accepts repo alias, task_type, and optional focus query
- [ ] Task types: debugging, feature_dev, code_review, documentation, exploration
- [ ] Task-type-aware content selection (e.g., debugging prioritizes error logs and issues)
- [ ] Token budget parameter with intelligent content trimming
- [ ] Priority-based content selection: critical context first, nice-to-have last
- [ ] Bundle sections: summary, relevant_code, related_issues, recent_changes, architecture
- [ ] Section-level truncation with "more available" indicators
- [ ] Response includes: bundle content, token_estimate, sections_included, sections_truncated
- [ ] Unit tests for different task types and budget constraints

## Notes

- Token estimation can be approximate (chars/4 is a rough heuristic)
- Debugging bundle: issues + error-related code + recent changes
- Feature dev bundle: architecture + related code + existing patterns
- Code review bundle: recent changes + coding conventions + related tests
- Consider allowing custom bundle templates in configuration
