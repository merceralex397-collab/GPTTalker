# OBS-003: Issue timeline

## Summary

Implement the get_issue_timeline MCP tool that provides a chronological view of issue activity per repository. Combine issue creation, status updates, and resolution events into a time-ordered timeline. This gives ChatGPT a narrative view of project health and issue progression.

## Acceptance Criteria

- [ ] get_issue_timeline MCP tool accepts repo alias and optional time range
- [ ] Timeline entries: issue created, status changed, note added, resolved
- [ ] Entries sorted chronologically (newest first by default)
- [ ] Each entry includes: timestamp, event_type, issue_title, severity, details
- [ ] Filter by severity level
- [ ] Summary header: total issues, open count, resolved count, critical count
- [ ] Pagination for long timelines
- [ ] Unit tests with sample issue history
