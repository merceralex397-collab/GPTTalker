# LLM-002: OpenCode adapter

## Summary

Build a dedicated adapter for the OpenCode coding agent server. Implement specific methods for session management (opencode_new_session, opencode_prompt, opencode_status) that map to OpenCode's API. This adapter handles the stateful nature of OpenCode sessions where a coding task may span multiple prompts.

## Acceptance Criteria

- [ ] OpenCode adapter class implementing the LLM service interface
- [ ] opencode_new_session: create a new coding session, return session_id
- [ ] opencode_prompt: send prompt to existing session, return response
- [ ] opencode_status: check session status (active, completed, error)
- [ ] Session ID tracking and validation
- [ ] Timeout handling for long-running code generation tasks
- [ ] Error mapping from OpenCode errors to hub error format
- [ ] Response includes: content, session_id, status, duration_ms
- [ ] Unit tests with mocked OpenCode API
