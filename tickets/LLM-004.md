# LLM-004: Embedding service adapter

## Summary

Build a dedicated adapter for embedding services that converts text into vector representations for semantic search. Support batch embedding operations for efficient processing of multiple texts, and integrate with the embedding pipeline used by the Qdrant-based context system.

## Stage

planning

## Status

todo

## Depends On

- LLM-001

## Acceptance Criteria

- [ ] Embedding adapter class implementing the LLM service interface
- [ ] Single text embedding: text → vector (list of floats)
- [ ] Batch embedding: list of texts → list of vectors
- [ ] Response includes: vectors, model_name, dimensions, duration_ms
- [ ] Configurable batch size to respect backend limits
- [ ] Automatic batching for large input lists
- [ ] Support for OpenAI-compatible /v1/embeddings endpoint
- [ ] Dimension validation (consistent across calls)
- [ ] Connection and timeout error handling
- [ ] Unit tests with mocked embedding responses

## Notes

- Common embedding models: nomic-embed-text, all-MiniLM, BGE
- Typical dimensions: 384, 768, 1536 — must match Qdrant collection config
- Batch size of 32-64 texts is typical for local models
- Consider caching embeddings for identical texts (hash-based)
