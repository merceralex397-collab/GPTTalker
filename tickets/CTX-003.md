# CTX-003: get_project_context tool

## Summary

Implement the get_project_context MCP tool that provides ChatGPT with a compact, relevant context bundle for a repository. Use hybrid retrieval combining keyword search and semantic search (Qdrant) to assemble a context package including project summary, architecture overview, key files, recent documents, and known issues. Support focused retrieval via a query parameter.

## Acceptance Criteria

- [ ] get_project_context MCP tool accepts repo alias and optional query
- [ ] Without query: returns general project overview context
- [ ] With query: returns context focused on the query topic
- [ ] Hybrid retrieval: combines keyword matches with semantic (Qdrant) results
- [ ] Context bundle includes: project_summary, architecture, key_files, recent_docs, issues
- [ ] Results ranked by relevance and deduplicated
- [ ] Response size bounded (configurable max tokens/chars)
- [ ] Graceful degradation: returns partial context if Qdrant unavailable
- [ ] "context_quality" indicator: full, partial, stale, unavailable
- [ ] Unit tests with mocked Qdrant and database
