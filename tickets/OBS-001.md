# OBS-001: Task history system

## Summary

Implement the append-only task history system that records all operations performed through the hub. Track task execution records, generated document records, indexing run records, and backend failure records. This provides the audit trail and observability foundation for understanding what the system has done.

## Acceptance Criteria

- [ ] Task record model: id, type, node, repo/service, start_time, end_time, result, error, trace_id
- [ ] Generated document record: id, task_id, repo_alias, file_path, content_hash, created_at
- [ ] Indexing run record: id, repo_alias, start_time, end_time, chunks_created, status
- [ ] Backend failure record: id, service_alias, error_type, error_message, timestamp, trace_id
- [ ] All records append-only (no updates or deletes)
- [ ] Automatic recording via middleware or decorator pattern
- [ ] Records include trace_id for correlation
- [ ] Database indexes for common query patterns (by type, repo, time range)
- [ ] Unit tests for record creation and query
