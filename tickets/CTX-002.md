# CTX-002: index_repo tool

## Summary

Implement the index_repo MCP tool that scans a repository for indexable content, chunks and embeds it, and stores the vectors in Qdrant. Indexable content includes README files, documentation, code summaries, recent commits, and known issues. Support both manual indexing (triggered by ChatGPT) and write-triggered indexing (after write_markdown operations).

## Acceptance Criteria

- [ ] index_repo MCP tool accepts repo alias and optional scope parameter
- [ ] Scans repo for indexable content: README, docs/, code files, commit messages
- [ ] Text chunking with configurable chunk size and overlap
- [ ] Chunks embedded via embedding service adapter
- [ ] Vectors stored in per-repo Qdrant collection with metadata
- [ ] Indexing status tracked in SQLite (last_indexed, chunk_count, status)
- [ ] Incremental indexing: only process changed files since last index
- [ ] Full re-index option that clears and rebuilds collection
- [ ] Progress reporting in response (files processed, chunks created)
- [ ] Write-triggered indexing hook (callable after write_markdown)
- [ ] Unit tests with mocked Qdrant and embedding service

## Notes
