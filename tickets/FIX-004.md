# FIX-004: Fix SQLite write persistence and uncommitted transactions

## Summary

DatabaseManager.execute()/executemany() never commit, and migrations/repositories perform DDL/DML via these methods outside the transaction() context manager. Schema changes and CRUD writes silently roll back on connection close. Either auto-commit in execute methods or ensure all callers use the transaction() context manager.
