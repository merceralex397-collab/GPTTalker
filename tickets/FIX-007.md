# FIX-007: Fix ripgrep search parser and implement search modes

## Summary

Two search defects: (1) src/node_agent/executor.py builds ripgrep command with -c (count mode) which outputs filename:count format, but the parser at lines 220-255 expects filename:line_number:match_count:content format, causing all searches to return zero matches. Remove -c flag or fix parser. (2) search_repo exposes only regex text search with no mode parameter. The spec requires mode='text|path|symbol' to support path and symbol search patterns. Add mode parameter and implement path/symbol backends.
