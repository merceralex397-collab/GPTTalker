# LLM-003: General and helper model adapters

## Summary

Build adapters for OpenAI-compatible API servers (llama.cpp, vLLM, Ollama, etc.) and helper model endpoints used for classification, summarization, and compression tasks. These adapters implement the LLM service interface and handle the specifics of each backend's API format.

## Acceptance Criteria

- [ ] OpenAI-compatible adapter: works with /v1/chat/completions endpoint
- [ ] Supports: model selection, temperature, max_tokens, system prompt
- [ ] Streaming support (optional, can buffer and return complete)
- [ ] Helper model adapter for lightweight tasks
- [ ] Helper methods: classify(text, categories), summarize(text, max_length), compress(text)
- [ ] Response format consistent with chat_llm tool output
- [ ] Auth header injection (Bearer token, API key)
- [ ] Connection error handling with meaningful messages
- [ ] Unit tests with mocked API responses

## Notes

- OpenAI-compatible API is a de facto standard — one adapter covers many backends
- Helper model can be a smaller/faster model on the same or different node
- Consider response format normalization across different backend implementations
- vLLM and llama.cpp may have slightly different response formats — handle gracefully
